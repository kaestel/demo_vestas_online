i:form

Form

Basic features

The extended form will handle events, and provides callbacks if you want to overwrite any default functionality.
It provides automatic validation on any <div class="field"> within the <form>.

Loading screens are also handled by the form, which invokes a loader when submitted, and removes it when response is received and processed.

Error- or success-messages from the server are automatically processed and shown on the page.

Methods

get (default) - regular get, adding parameters to url.

post - regular post.

script - script injection in head (jsonp method).

Variations

Form submit types

type:manual (default) - submit buttons and [ENTER] submits the form via AJAX request.

type:auto - submits the form onload via AJAX request. Can be used to pre-fill search result. Additional AJAX-based submits can be done manually.

type:html - submit buttons and [ENTER] submits the form as a regular HTML form.

Send data options

send:parameters (default) - send data as parameters. Ex.: varname=var

send:json - send data as JSON string. Ex.: {"varname":"var"}

send:jdata - send data as JSON in jdata parameter. Ex.: jdata={"varname":"var"}

Response handling

Parse response to template

If you want a JSON response to be parsed directly into a template, just add a .template element inside your form.

<form action="servlet.json" method="get" class="i:form type:auto"> <table> <tr class="header"> <th class="id">ID</th> <th class="title">Title</th> </tr> <tr class="template"> <td>{agreement_id}</td> <td><a href="um-agreement.html?agreement_id={agreement_id}">{agreement_title}</a></td> </tr> </table> </form>

Forward URL

Add an input with name="response_success_forward_url" if you want to forward the user to another url, when the form submits successfully (submit returns success).

If you want to transfer parameters from the response to the forward url, add an input with name="response_success_forward_parameters" and state your parameters here. Use comma to separate parameteres.

Callback functions

form.submitted

When the form is validated and ready to submit, this callback function will be invoked. This is a predefined function, which can be overwritten if custom submit handling is required.

form.changed

When the form is changed, meaning, when any field of the form throws an onchange event, this callback function is invoked. Declare this function to handle the event. Can be used to change button state when the form content is changed.

form.updated

When the form is changed, meaning, when any typing, clicking or selecting in fields, this callback function is invoked. Declare this function to handle the event. Can be used to change button state when the form content is changed.

form.validationFailed

If attempt is made to submit the form, but validation fails, this callback function is invoked. Can be used to create customized error handling.
