Forms and Buttons

Forms

Forms, regardless of wether they are regular HTML forms or AJAX forms with JSON response, should be written in the same way - as a plain regular HTML form using the specified HTML syntax and the i:form initializer class.

See i:form for more information about form variations.

<form name="logon" action="#URL#" method="#METHOD#" class"i:form"> <fieldset> <div class="field string optional"> ... </div> <div class="field string required"> ... </div> </fieldset> <ul class="actions"> ... </ul> </form>

You can see a demo on the form demo page.

<form>

A <form> with action, method and name. The action-attribute specifies primary form url, the method-attribute specifies the way the form parameters are submitted.

Use <fieldset>s to group sets of fields.
NOTE: You should NEVER be applied any styles to the <fieldset> - it is only used for structural purposes.

<div class="field">

The form is composed of a set of fields, individually wrapped in a <div class="field">, specifying the field types. Inputs, selects and textareas wrapped in correct field-types are automatically extended with clientside validation with hints and error handling.

<div class="field string optional"> <label for="name">Navn</label> <input type="text" name="name" id="name" value="" /> <p class="error">Error message</p> <p class="hint">Hint message</p> </div>

A field must contain a <label> and an <input>/<select>/<textarea>.

<label>s for-attribute must match <input>, <select> or <textarea> ID-attribute.

<input>s must have a valid type-attribute.

<input>s, <select>s and <textarea>s must have a valid name-attribute.

The field class is used to specify additional info about the field, such as field-type and whether the field is optional or required. The field must have a valid type-declaration to be initialized.

Field types

  • string for input type text. Validation requires min. 2 chars text in input.
  • email for input type email. Validation requires valid formatted email.
  • text for textareas. Validation requires min. 3 chars text in textarea.
  • select for selects. Validation requires selected option with value.
  • checkbox for checkboxex. Validation requires selection.
  • radio for radiobuttons. Validation requires one selection in radio-group.
  • date for date fields. Validation requires valid date.

If a field is required, the required must be added.

A .field can have a supporting <p class="error"> and/or <p class="hint"> to be used with JavaScript validation.

<ul class="actions">

Buttons are always placed in a <ul class="actions">, preferably outside <fieldset>s, but always inside the <form> they relate to.

<ul class="actions"> <li class="cancel"><a href="#" class="button">Cancel</a></li> <li class="submit"><input type="submit" value="#name#" class="button primary" /></li> </ul>

Each button is placed inside its own LI and should have a defining classname. Common <li> classnames are submit, cancel, reset, back, info, add, edit, new, close. Try to reuse existing names if possible - but the primary objective is to make the HTML easy to read.

A button can be either a <a class="button"> or an <input class="button">.

The submit button should always be an <input class="button" type="submit">.

Button types

  • button - plain neutral button
  • button primary - primary action, typically a submit
  • button tool - tool button with an icon, like share page
  • button disabled - disabled button

Adding a disabled class to a button will make it unclickable and grayed out. The disabled class can be added to any other button type.
