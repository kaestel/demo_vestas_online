Applications

An application is the view from an Application that is shown on the Add Application page. Applications have two states: Installed and ready to install and the HTML differs a bit between the two states.

The overall structure for showing a list of apps is:

<ul class="apps"> <li class="app">...</li> <li class="app">...</li> <li class="app">...</li> <li class="app">...</li> </ul>

The HTML for an installed app is:

<li class="app"> <img src="/img/bg_app_logo.png" alt="Issue Manager" /> <h3>Issue Manager</h3> <p>Nunc feugiat, lorem vitae placerat auctor, nibh elit dignissim orci.</p> <ul class="actions"> <li class="info"><a href="/templates/add_application_view.html" class="button">Get info</a></li> <li class="add"><a class="button disabled">Installed</a></li> </ul> </li>

The HTML for an app that is ready to install is:

<li class="app"> <img src="/img/bg_app_logo.png" alt="Document Sharing"/> <h3>Document Sharing</h3> <p>Nunc feugiat, lorem vitae placerat auctor, nibh elit dignissim orci.</p> <ul class="actions"> <li class="info"><a href="/templates/add_application_view.html" class="button">Get info</a></li> <li class="add"> <form action="/util/MDAppAdd" method="post" name="add_DocumentSharing"> <input type="hidden" name="appid" value="DocumentSharing" /> <input type="hidden" name="appurl" value="http://vestas.local/testapp" /> <input type="hidden" name="appname" value="My new app" /> <input type="submit" value="Add App" class="button primary" /> </form> </li> </ul> </li>

The "appname" and "appurl" form values are used to inject a new item in the main navigation after adding an application and "appid" is used to identify the application
