Code guidelines

General HTML guidelines

The HTML templates have been structure and designed to work across different devices to ensure forward and backwards compatibility. The functionality layer (JavaScript) and the design layer (CSS) are kept separate from the HTML, so they can be changed depending on which device is visiting the page.

It means the HTML should be clean, structural and semantic. And it means:

  • NEVER use inline CSS! No style attributes. No style tags.
  • NEVER use inline JavaScript! No onclick attributes. No javascript links. No script tags.

If you add inline CSS or JavaScript it cannot be varied depending on the device and it cannot be updated centrally. Thus we loose ability to build a cross-device experience with a consistant user experience.
