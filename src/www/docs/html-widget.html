Widgets

A widget is the view from an Application that is shown on the Jump Board. Widgets can take up either 100% or 50% of the page width. Widgets are 100% width per default and you can make it 50% width by adding a .half class to the list item.

The widgets have an overall HTML structure that needs to be followed, but the content in div.body is not general and currently needs custom CSS for each widget. In the first example below you will notice that each li.widget also as a secondary class name which is used for the custom styling.

The overall HTML structure for showing widgets is:

<ul class="widgets i:widgets"> <li class="widget name_of_widget">100% width widget</li> <li class="widget half name_of_second_widget">50% width widget</li> <li class="widget half name_of_third_widget">50% width widget</li> <li class="widget issue_manager">100% width issue manager widget</li> </ul>

The contents of a widget consists of a div.header and a div.body. Both div tags are mandatory.

li.widget div.header

The widget header consists of a widget name in a h2 tag and a ul.actions list. The ul.actions list contains a .launch list item and a ul.settings list as shown below:

<div class="header"> <h2>Application name</h2> <ul class="actions"> <li class="settings"><h3>Settings</h3> <ul> <li class="edit"> <h4><a href="#">Edit Settings</a></h4> <div class="notification">New version!</div> </li> <li class="remove"> <form action="/util/MDAppRemove" method="post" name="remove_dummy"> <input type="hidden" name="appid" value="dummy" /> <input type="submit" class="button" value="Remove App" /> </form> </li> <li class="about"><h4><a href="#">About this App</a></h4></li> </ul> </li> <li class="launch"><a href="#">Launch app</a></li> </ul> </div>

li.widget div.body

The widget body can contain custom html so no example is necessary.
