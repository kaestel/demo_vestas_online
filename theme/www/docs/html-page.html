Page structure

Quick overview

The basic page structure is composed of an outer #page, which contains all other elements. A #header containing .client, .user and #navigation. #subnavigation, #content, #feedback. #footer. Each element, except for #page and #header, can be omitted and the remaining nodes will collapse the void, to maintain the intended page appearance.

Simplified, it looks like this:

<div id="page" class="i:page"> <div id="header"> <div class="client">...</div> <div class="user">...</div> <div id="navigation">...</div> </div> <div id="subnavigation">...</div> <div id="content" class="#scope#"> Individual content goes here </div> <div id="feedback" class="i:feedback">...</div> <div id="footer">...</div> </div>

Structural elements

#page

<div id="page" class="i:page">

This tag wraps the entire page, ensures correct position on screen when using overlays and such. Furthermore this tag is the basis for initialization of the page controller. Thus, this tag and it's class cannot be omitted.

The i:page-class is an initalizer class. This class invokes the primary page controller on the #page-node. The page controller initalizes the navigation, user menu, hints and overlays. It also extends the #page-node with properties and references used by other initializers and utility functions.

#header

<div id="header">

This node is fixed to the top of the browsers viewable area. It contains some important page elements. The .client (client name and back link), the .user (current user information and account maintenance dropdown menu) and the #navigation (primary navigation dropdown).

.user

<div class="user"> <h3>#user-name#</h3> <div class="notification">#optional notification count#</div> <ul> <li> <h4><a href="#user-account-link#">#user-account-option#</a></h4> <div class="notification">#optional notification text#</div> </li> <li class="signout"> <h4><a href="/util/SECLogoff">Sign Out</a></h4> </li> </ul> </div>

H3 for user name.
UL for menu options.
Add as many LI's as needed.
Each LI requires H4 for item title.
Use regular A with HREF for links.
Each LI can have an optional .notification.

Keep LI.signout as last element in UL.

This element is extended with JavaScript. The HTML must be as specified or the JavaScript might not work as intended.

#navigation

<div id="navigation"> <h3>Launch App</h3> <ul> <li> <h4><a href="#link#" target="_blank">#name#</a></h4> <div class="notification">#optional notification#</div> </li> <li class="add"><h4><a href="#link#">Add Application</a></h4></li> </ul> </div>

H3 for Navigation name.
UL for menu options.
Add as many LI's as needed.
Each LI requires H4 for item title.
Use regular A with HREF for links.
Each LI can have an optional .notification.

Keep LI.add as last element in UL.

This element is extended with JavaScript. The HTML must be as specified or the JavaScript might not work as intended.

#subnavigation

<div id="subnavigation"> <ul> <li class="selected"><h4><a href="#link#">#name#</a></h4></li> <li> <h4><a href="#link#">#name#</a></h4> <div class="notification">#optional notification#</div> </li> </ul> <ul class="actions">...</ul> </div>

H3 for Navigation name.
UL for menu options.
Add as many LI's as needed.
Each LI requires H4 for item title.
Use regular A with HREF for links.
Each LI can have an optional .notification.
LI.selected sets selected state for current LI.
UL.actions for page specific service-navigation.

#subnavigation .actions

<ul class="actions"> <li class="i:sharepage"><a href="#link#" class="button tool">Share this page</a></li> </ul>

The #subnavigation also contains a page-specific "share this page" feature. The LI has an i:sharepage-class initalizer class, which loads and shows the sharing panel and sends sharing requests to the server.

i:sharepage uses the value of the A-href to load the sharing panel. It submits the sharing as specified in the loaded HTML form. See FORM documentation for details.

#content

<div id="content" class="#scope#">...</div>

This is where your content generally goes. Unless you are extending the overall page functionality you should not add code anywhere but here.

Use the #content-node class attribute to specify your scope. If your page layout fails, check if your #content class is set correctly.

#feedback

<div id="feedback" class="i:feedback"> <h3><a href="#link#">Feedback</a></h3> </div>

The #feedback is a side-wide feedback feature. The #feedback-node has an i:feedback-class initalizer class, which loads and shows feedback panel and sends feedback to the server.

i:feedback uses the value of the A-href to load the feedback panel. It submits the feedback as specified in the loaded HTML form. See FORM documentation for details.

#footer

<div id="footer"> <div class="tagline">Wind. It means the world to us.&trade;</div> </div>

Places at the bottom of the page. Should always be the last element in #page. Content is actually hidden behind a graphic.

General conventions

.notification

<div class="notification">...</div>

The .notification node can be inserted when specified.
It is always optional, and the surrounding HTML will collapse appropriately if .notification is left out.
As a conventions the .notification is always placed right after the H4 it relates to.
The node will be manipulated by JavaScript and the HTML must be as specified to work properly.

Add .disabled class to .notification node to invoke disabled state.
